# Walk-Back: Phase 1/2 Bug + Revalidation — 2026-05-18

## The bug

`_scripts/extract_features_birdnet.py` (commit `080bd9a` and earlier) used
`result.embeddings_masked` (BOOL validity mask, dtype=bool) instead of
`result.embeddings` (FLOAT32 actual embedding data) from the BirdNET API.

Consequence: every recording's "1024-dim embedding" was actually a single
scalar value (= `n_valid_segments / max_segments`) broadcast to all 1024
columns. The values were constrained to N/21 fractions for integer N.

So the Phase 1/2 pipeline did NOT run on 1024-dim BirdNET features. It ran
on a single scalar per recording.

## What broke + what survived after fixing

Fixed extraction at commit (pending) writes to
`acoustic_features/birdnet_embeddings_v2.parquet` (1344/1344 ok, real
1024-dim float32 with L2 norms 10-21, 1339/1344 unique rows).

### BROKE: dialect-cluster labels

`dialect_clusters_v1.parquet` (used by the Stan model for cell partial-
pooling) was extracted from broken embeddings. We re-ran UMAP+HDBSCAN on
real embeddings → `dialect_clusters_v2.parquet`:

| Species | v1 clusters | v2 clusters | ARI | NMI |
|---------|-------------|-------------|-----|-----|
| Carolina chickadee | 2 | 7 | **0.002** | 0.025 |
| marsh wren | 2 | 6 | **−0.010** | 0.012 |
| song sparrow | 6 | 3 | **−0.009** | 0.024 |
| white-crowned sparrow | 4 | 8 | 0.047 | 0.087 |

**ARI ≈ 0 across all species means v1 and v2 cluster partitions are
statistically independent.** The v1 "14 dialect clusters" finding has zero
relationship to the partition we'd extract from real embeddings.

The Phase 1 wrap-up claim of "14 dialect clusters discovered via unsupervised
UMAP+HDBSCAN on BirdNET embeddings" is not reproducible. The clusters that
exist in real BirdNET space are different in count, different in size, and
have different memberships.

## Where Phase 1/2 wrap was wrong

1. **"14 dialect clusters" specific finding**: not reproducible. Real BirdNET
   space gives 24 total clusters with ARI ≈ 0 to v1. Drop this claim.

2. **"Cell partial-pooling on BirdNET-embedding distance generalizes across
   3 axes"**: now 2.5/3. Recordist-split longtail subgroup FAILS at 5%
   threshold on v2 (was PASS on v1). The robustness across this axis isn't
   what we said.

3. **"+4.7 pp over Sainburg-2020 baseline"**: now +3.4 pp. Still positive,
   smaller margin. Update the number.

4. **The interpretation that the cell partition reflects "real bird-song
   dialect structure"**: not supported. ARI ≈ 0 between v1 and v2 partitions
   means cluster MEMBERSHIPS were essentially noise. The Stan model picked
   up frac_cell ≈ 0.19 in BOTH partitions, which more likely means cell
   partial-pooling adds variance because it adds free parameters, not
   because the cells correspond to biological dialects.

## Where Phase 1/2 wrap was right

1. **Spectral signal is robust** to the embedding fix. Same frac_cell. PASS
   at 5% threshold. 5/5 folds, ecoregion swap, top recordists.

2. **Structural/syntactic null** holds. Phrase-template DTW (Angle 2 today)
   adds another operationalization where rhythmic differences are null at
   regional level. The structural null isn't operationalization-fragile.

3. **PARTIAL disposition** is correct — the spectral side carries signal,
   the structural side doesn't.

## Open concern — needs further test

The fact that cell-partial-pooling frac_cell ≈ 0.19 survives the v1→v2
cluster reassignment (ARI ≈ 0) raises the question: would frac_cell still
be ≈ 0.19 if we used RANDOM labels? If yes, the cell layer is fitting
degrees-of-freedom, not real dialect structure. If no, v2 clusters genuinely
capture additional variance.

This test is the next thing to run. Outcome determines whether the spectral
signal is "real cluster structure" or "any partition adds variance."

## Honest disposition

The bug existed. The dialect-cluster specifics in the wrap-up are not
reproducible. But the cell-partial-pooling spectral signal that defined
Phase 1's PASS verdict does reproduce on real embeddings at comparable
magnitude. The "PARTIAL" disposition stands; the interpretation around
dialect-structure needs revising.
